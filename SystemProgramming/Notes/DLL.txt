DLL - Dynamic Link Library

Библиотека кода - отдельные файлы с кодами дополнительных функций\классов
Деляться на 
1 - библеотеки статической компоновки (как в С++) - содержимое файлов 
    добавляется как текст к нашей программе, после чего компилируется 
    собранный файл.
2 - библиотека динамической компановки - скомпилированные файлы , в которых код готов 
    к исполнению , другие программы могут использовать этот код для запуска в составе своих прирложений.

1: 
 "+" файл программы один, все необходимое находиться в нём.
 "-" Для запуска программы необходимо наличие всех используемых Dll.
 "+" меньшая зависимость от OS , для адаптации достаточно перекомпилировать программу.

2: 
 "-" Для запуска программы необходимо наличие всех используемых Dll.
 "+" библиотека одна,все файлы могут к ней обращаться , ее коды никак не изменяют размер программ.
 "-" исполнимость сильно зависит от OS, по этому DLL практически невозможно переносить между ними

 ___
в виде DLL реализуют системные функции  - они поставляются с OS, регистрируются в папках общего доступа,
скомпилированы в полной адаптации с OS , сохраняя названия и имена функций для использования в других программах.
Совокупность инструментов OS для прикладных программ называют API
(Application - Program Interface)

 !! расширение файлов .dll имеют как реально исполнимые файлы (те же .exe файлы, только без main() - без точки входа),
 так и библеотека классов .NET , которые являются промежуточным (IL) кодом и исполняются JIT платформы.

 Marshal - Маршализация - процесс взаимодействия (обменна данными) между управляемым кодом и неуправляемым кодом, interoperability
